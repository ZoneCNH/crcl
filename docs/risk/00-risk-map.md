# 风险地图

## 结论

CRCL 的风险不是单一股价波动，而是五类结构风险：

1. 储备信用风险。
2. 利率和规模错配风险。
3. 分销成本与渠道依赖风险。
4. 监管重写商业模式风险。
5. 平台化失败与估值回落风险。

结构风险是会改变投资逻辑的风险，白话就是不是跌一跌，而是故事本身变了。

## 风险优先级

| 风险                         | 严重级别 | 原因                      |
| ---------------------------- | -------- | ------------------------- |
| 储备异常或赎回信任受损       | P0       | 信用层失效后估值无意义    |
| 连续净赎回叠加市占率下降     | P0       | 说明竞争力和需求同时转弱  |
| 监管限制收益分享或第三方激励 | P0       | 直接影响分销经济性        |
| RLDC margin 持续下滑         | P1       | 规模增长留不住利润        |
| Other revenue 低于预期       | P1       | 平台化倍数下修            |
| Coinbase 议价权增强          | P1       | Base 增长可能变成渠道成本 |
| 利率下行且 USDC 不增长       | P1       | 利差现金流被压缩          |
| 解禁、增发、内部人减持       | P2       | 放大股价波动              |

## 风险传导链

```text
储备异常
  -> USDC 折价
  -> 赎回扩大
  -> 市占率下降
  -> Reserve income 下滑
  -> RLDC 下滑
  -> 估值倍数压缩
```

```text
监管限制第三方激励
  -> 交易所 / 钱包分销模式重估
  -> Distribution and transaction costs 改变
  -> RLDC margin 不确定
  -> 平台化倍数延后
```

## 风险矩阵

| 维度   | 观察指标                           | 触发动作              |
| ------ | ---------------------------------- | --------------------- |
| 信用   | USDC/USD、储备报告、赎回通道       | 立即检查失效条件      |
| 现金流 | USDC 平均流通量、储备收益率、RLDC  | 重算收入模型          |
| 成本   | 分销成本率、Coinbase 依赖          | 调整 RLDC margin 假设 |
| 监管   | GENIUS Act 细则、OCC、Treasury     | 重新判断商业模式      |
| 竞争   | USDC 市占率、USDT、银行稳定币、RWA | 调整长期倍数          |
| 筹码   | 解禁、Form 4、short interest、IV   | 调整风控而非基本面    |

short interest 是做空比例，白话就是市场上押注股价下跌的仓位。
IV 是 implied volatility，白话就是期权市场预期未来波动有多大。

## 信用层触发条件（独立于 competition score）

以下任一满足，立即触发信用层协议（见 `risk/02-failure-conditions.md`），不受竞争得分影响：

| 触发条件 | 参数 | 数据来源 |
| -------- | ---- | -------- |
| C-TRIGGER-A：USDC 市占率 MoM 下降 | > 3%，连续 2 个月 | CoinGecko / DefiLlama 月末快照 |
| C-TRIGGER-B：USDC 绝对规模 MoM 下降 | > 5%，连续 2 个月 | Circle 官方数据 / The Block |
| C-TRIGGER-C：单周净赎回超季度末余额比例 | > 1.5%，连续 2 周 | The Block 净铸造/销毁数据（注：不直接用链上铸造/销毁事件，使用综合统计数据）示例：Q4末 $75.3B → 阈值 ≈ $11.3亿/周，每季末重置基数 |

**竞争层与信用层优先级**：`competition_score`（竞争层）与信用层触发使用独立机制。信用层触发（C-TRIGGER-A/B/C）优先于竞争层判断，不受 `competition_score` 当前数值影响。两套系统并行运行，竞争得分持续计算，但信用层一旦触发，立即执行 `risk/02` 协议。

## 信用评分机制

信用评分是储备信用层的量化辅助，独立于 competition_score 运行，不影响情景框架切换。

### 信用评分构成

```text
credit_score（0-100）=
  40% 储备透明度与合规性
  30% USDC 锚定稳定性
  20% 赎回流动性
  10% 管理层信息质量
```

信用评分的目的是早于 C-TRIGGER-A/B/C 感知到储备信用变化，提供提前观察信号。

### 各维度评分标准

#### 储备透明度（40 分）

| 分数 | 条件 |
| ---- | ---- |
| 35-40 | 月度储备披露正常，审计无异常，BlackRock USDXX 基金数据更新 |
| 25-34 | 披露延迟 1-3 周，或审计师发表保留意见 |
| 10-24 | 披露延迟超过 30 天，或储备构成发生未解释变化 |
| 0-9 | 披露暂停，或储备报告出现重大异常 |

#### USDC 锚定稳定性（30 分）

| 分数 | 条件 |
| ---- | ---- |
| 28-30 | USDC/USD 偏差 <0.05%，持续 30 天 |
| 20-27 | USDC/USD 偏差 0.05-0.1%，或单日短暂到 0.15% |
| 10-19 | USDC/USD 偏差 >0.1%，持续超过 24 小时 |
| 0-9 | USDC/USD 折价 >0.3%，赎回受限信号出现 |

#### 赎回流动性（20 分）

| 分数 | 条件 |
| ---- | ---- |
| 18-20 | 大额赎回（>$5 亿/天）正常执行，无延迟 |
| 12-17 | 赎回渠道正常，但有延迟报告或小幅质疑 |
| 5-11 | 赎回渠道出现公开延迟，或合作方报告摩擦 |
| 0-4 | 赎回限额、暂停、公开异常 |

#### 管理层信息质量（10 分）

| 分数 | 条件 |
| ---- | ---- |
| 9-10 | 财报和电话会披露完整，答分析师追问清晰 |
| 6-8 | 部分问题回避，但定量口径一致 |
| 3-5 | 多次回避关键追问，口径有明显前后矛盾 |
| 0-2 | 主动撤回信息，或披露异常 |

### 信用评分阈值与动作

| 分数范围 | 信用状态 | 动作 |
| -------- | -------- | ---- |
| 80-100 | 信用健康 | 正常研究，无额外操作 |
| 60-79 | 信用观察 | 提高监控频率，关注趋势方向 |
| 40-59 | 信用预警 | 降低仓位因子上限，等待下一个观察窗口 |
| 20-39 | 信用警报 | 暂停加仓，准备快速减仓计划 |
| 0-19 | 信用危机 | 执行清仓或极轻仓，不等 C-TRIGGER 计数完成 |

### 与 C-TRIGGER 的关系

- credit_score 是早期感知工具，C-TRIGGER-A/B/C 是执行触发工具。
- 当 credit_score 低于 40 时，C-TRIGGER 计数加倍关注（不等到第 2 周才复核）。
- 当 C-TRIGGER 触发时，credit_score 必然在 40 以下（否则 C-TRIGGER 不该触发）。
- credit_score 高（>80）但 C-TRIGGER 被触发，说明数据存在分歧，需要额外核查来源。

### 信用评分更新频率

- 储备透明度维度：月度披露后更新
- USDC 锚定维度：每日检查（P0 指标）
- 赎回流动性维度：有赎回事件或市场压力时更新
- 管理层信息质量维度：财报和电话会后更新

## 风险处理纪律

P0 风险出现时，先判断信用和商业模式。
不要先看估值便宜不便宜。

P1 风险出现时，先看是否连续。
单期成本上升需要听管理层解释，连续两个季度恶化才降级。

P2 风险出现时，先看是否和基本面共振。
单独解禁不一定破坏长期逻辑，但会放大短期回撤。
