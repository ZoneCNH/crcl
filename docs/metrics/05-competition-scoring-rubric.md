# 竞争评分细化规则

## 结论

本文件为 competition score 的每个维度提供逐档打分标准，目标是消除"同数据不同分"的主观偏差。
打分者依据本文件填写 `metrics/04-competition-dashboard.md` 的 Competition Score 表格。

## 评分总说明

### 使用规则

| 规则       | 内容                                                                              |
| ---------- | --------------------------------------------------------------------------------- |
| 打分周期   | 每周一次，与 `04-competition-dashboard.md` 同步更新                               |
| 分数档位   | 每个维度只能取 0 / 25 / 50 / 75 / 100；不允许打 60、70 等中间值                  |
| 档位选择   | 实际数据落在两个档位中间时，向对 CRCL 不利的方向取整（保守原则）                  |
| 数据依据   | 每个维度必须在"证据"列写出具体数值，不允许凭印象打分                              |
| missing_info | 数据缺失时维度分数填 `missing_info`，不按 50 分处理；缺失维度不参与加权计算，总分按实际可得维度权重归一后输出 |
| 版本规则   | 本文件版本号格式 `YYYY-MM-DD`；每次修改任何档位阈值时更新版本并记录变更原因       |

当前版本：**2026-05-11**

### 打分公式

```text
competition score =
  25% × D1（USDC 市占率趋势）
  + 15% × D2（USDT 相对威胁）
  + 20% × D3（分销渠道议价）
  + 15% × D4（收益型稳定币蚕食）
  + 10% × D5（链生态健康）
  + 10% × D6（监管壁垒）
  + 5%  × D7（平台化验证）
```

### 总分结论对照

| 总分   | 结论         | 动作                        |
| -----: | ------------ | --------------------------- |
| 75-100 | 竞争结构强化 | 可上调平台化权重            |
| 55-74  | 中性偏强     | 维持框架，等财报验证        |
| 40-54  | 中性偏弱     | 暂停上调估值倍数            |
| 0-39   | 竞争结构恶化 | 重算增长率、RLDC 和长期倍数 |

## D1：USDC 市占率趋势（权重 25%）

### 定义

衡量 USDC 在稳定币总市场中的相对份额变化趋势，以及与 USDT 的差距演变。绝对市占率水平不计分，只看方向和速度。

### 数据来源

| 指标                    | 来源                     | 取数时间          |
| ----------------------- | ------------------------ | ----------------- |
| USDC market share（%）  | DefiLlama + CoinGecko    | 周四 UTC 23:59    |
| USDC / USDT ratio       | 自算（USDC 市值 ÷ USDT 市值） | 同上           |
| USDC circulating supply | Circle Transparency      | 同上              |

两个来源偏差 >1 ppt 时取较低值，并在证据栏标注偏差。

### 打分标准

| 分数 | USDC market share 趋势（4周滚动）            | USDC / USDT ratio 趋势 |
| ---: | -------------------------------------------- | ----------------------- |
|  100 | 连续 4 周上升，累计升幅 ≥1.0 ppt             | 同期上升 ≥0.02          |
|   75 | 连续 2-3 周上升，或 4 周内累计升幅 0.5-1.0 ppt | 同期上升或持平          |
|   50 | 4 周内变化在 ±0.3 ppt 以内（横盘）；总盘扩张且 USDC 绝对值增长 | 变化 ±0.01  |
|   25 | 连续 2 周下降，或 4 周内累计下降 0.3-1.0 ppt | 同期下降                |
|    0 | 连续 4 周下降，或单月累计下降 ≥1.0 ppt       | 同期下降且降幅 ≥0.03    |

两项指标方向不一致时（如市占率升但 ratio 降），取较差项对应分数。

---

## D2：USDT 相对威胁（权重 15%）

### 定义

衡量 USDT 的规模扩张和离岸渗透对 USDC 生存空间的压缩程度。威胁越小，分数越高。
USDT 获得合规化或大型机构背书视为威胁升级事件，直接触发分数下调。

### 数据来源

| 指标                   | 来源                     | 取数时间       |
| ---------------------- | ------------------------ | -------------- |
| USDT dominance（%）    | DefiLlama                | 周四 UTC 23:59 |
| USDC / USDT ratio      | 自算                     | 同上           |
| USDT 离岸扩张事件      | Tether 官网 / 新闻稿     | 当周实时       |

### 打分标准

| 分数 | USDT dominance 趋势（4周滚动）             | 事件条件                             |
| ---: | ------------------------------------------ | ------------------------------------ |
|  100 | 连续 4 周下降，累计降幅 ≥2.0 ppt           | 当月无新 USDT 离岸合作或合规化公告   |
|   75 | 下降 1.0-2.0 ppt，或连续 2-3 周下降        | 无新增大型交易所 USDT 独家协议       |
|   50 | 4 周内变化在 ±1.0 ppt 以内（横盘）         | 无重大事件                           |
|   25 | 连续 2 周上升，或 4 周内累计上升 1.0-2.0 ppt | 出现 USDT 新大型交易所合作或试点合规 |
|    0 | 连续 4 周上升，累计上升 ≥2.0 ppt           | Tether 获得明确监管许可或主权级采用  |

当 USDT dominance 和事件条件对应不同分数时，取较低值。

---

## D3：分销渠道议价（权重 20%）

### 定义

衡量 Circle 在分销成本博弈中的留存能力。核心指标是 RLDC margin（Revenue Less Distribution Costs 除以 Reserve income）。分销成本侵蚀越多，分数越低。

### 数据来源

| 指标                          | 来源                         | 更新频率 |
| ----------------------------- | ---------------------------- | -------- |
| RLDC margin（÷ Reserve income）| Circle SEC filing（自算）    | 季度     |
| 分销成本率（Distribution costs ÷ Total revenue） | Circle SEC filing | 季度 |
| Base USDC supply / USDC 总供应 | Dune Analytics / TokenTerminal | 每周 |

RLDC margin 仅季度财报后更新；本维度每周打分时，非财报周沿用上一季度 RLDC margin 值，并在证据栏标注"沿用 Qx"。

### 打分标准

| 分数 | RLDC margin（主，÷储备收入） | 分销成本率环比变化 | Base USDC 占比（占总 USDC 供应） |
| ---: | --------------------------- | ------------------ | -------------------------------- |
|  100 | ≥42%                        | 改善 ≥1.0 ppt      | ≤28%                             |
|   75 | 40-42%                      | 变化在 ±1.0 ppt 内 | 28-35%                           |
|   50 | 38-40%                      | 上升 1.0-2.0 ppt   | 35-40%                           |
|   25 | 36-38%                      | 上升 2.0-4.0 ppt   | 40-45%                           |
|    0 | <36%                        | 上升 ≥4.0 ppt 或监管限制第三方激励触发 | ≥45% 且 RLDC margin 同期下滑 |

三项指标取最低对应分数。RLDC margin 是主锚：若 margin <38%，无论其他两项，本维度最高 25 分。

---

## D4：收益型稳定币蚕食（权重 15%）

### 定义

衡量 USDe / sUSDe 及 RWA 类产品（BUIDL / USDY / USYC）对 USDC 闲置持有人的资金虹吸程度。关键触发是收益型产品 AUM 增长与 USDC 净赎回同时发生。

### 数据来源

| 指标                              | 来源                      | 更新频率 |
| --------------------------------- | ------------------------- | -------- |
| USDe / sUSDe AUM                  | DefiLlama（Ethena）       | 每周     |
| BUIDL + USDY + USYC 合计 AUM      | RWA.xyz（交叉 DefiLlama） | 每周     |
| USDC net mint / redeem（当周）    | Circle Transparency       | 每周     |
| Aave / Compound USDC borrow 利用率 | Dune Analytics           | 每周     |

收益型产品总 AUM = USDe + BUIDL + USDY + USYC 合计值。

### 打分标准

| 分数 | 收益型产品总 AUM 周环比变化 | USDC net mint/redeem 状态       | Borrow 利用率 |
| ---: | --------------------------- | ------------------------------- | ------------- |
|  100 | <5%                         | 净铸造（>+5 亿美元）            | ≥60%          |
|   75 | 5-10%                       | 净铸造或零附近（±5 亿美元内）   | 40-60%        |
|   50 | 10-20%                      | 净铸造或单周小额净赎回 <5 亿美元 | 30-40%       |
|   25 | ≥20% 且单周净赎回 5-20 亿美元 | 净赎回 5-20 亿美元             | 20-30%        |
|    0 | 连续 4 周 ≥15%，且净赎回连续 2 周 ≥5 亿美元 | 净赎回 ≥20 亿美元 | <20% |

单周净赎回 ≥20 亿美元为单项触发条件，直接判 0 分，不等 AUM 条件满足。

---

## D5：链生态健康（权重 10%）

### 定义

衡量 USDC 链上实际使用质量，剔除停泊资金，看真实支付和借贷活跃度。供应增长不代表使用质量，必须结合 adjusted transfer volume 和借款活跃度判断。

### 数据来源

| 指标                              | 来源                         | 更新频率 |
| --------------------------------- | ---------------------------- | -------- |
| Adjusted transfer volume（4周 MA） | CoinMetrics / Dune           | 每周     |
| Active addresses（当周）          | Dune / Santiment             | 每周     |
| Aave / Compound USDC borrow 利用率 | Dune Analytics              | 每周     |
| USDC by chain（Ethereum + Solana 占比） | DefiLlama             | 每周     |

Adjusted transfer volume 取 4 周滚动均值，避免单周波动误判。

### 打分标准

| 分数 | Adjusted transfer volume 4周MA 环比 | Active addresses 环比 | Borrow 利用率 |
| ---: | ----------------------------------- | ---------------------- | ------------- |
|  100 | 上升 ≥5%                            | 上升 ≥5%              | ≥65%          |
|   75 | 上升 2-5%                           | 上升 0-5% 或持平       | 50-65%        |
|   50 | 变化在 ±2% 以内                     | 持平（±2%）            | 35-50%        |
|   25 | 下降 2-5%                           | 下降 2-8%              | 20-35%        |
|    0 | 下降 ≥5%，或连续 4 周下降           | 下降 ≥8%，且 Ethereum + Solana 占比降至 <50% | <20% |

三项各自对应分数，取最低值。Base 链占比上升但 Ethereum + Solana 占比不升时，不算链生态改善，维持或下调。

---

## D6：监管壁垒（权重 10%）

### 定义

衡量监管框架对 CRCL 竞争护城河的加固或侵蚀程度。包含两个方向：GENIUS Act 等有利规则的落地进展，以及银行稳定币和 tokenized deposits 的制度性威胁。

### 数据来源

| 指标                          | 来源                          | 更新频率   |
| ----------------------------- | ----------------------------- | ---------- |
| GENIUS Act 立法进展           | Congress.gov / OCC / Treasury | 实时       |
| OCC / FDIC 负面草案或裁决     | OCC / FDIC 新闻稿             | 实时       |
| 银行稳定币企业客户数          | 银行 IR / OCC / FDIC 公告     | 实时       |
| tokenized deposits 大行试点   | FDIC / 银行公告               | 实时       |

无事件周，本维度维持上周评分，在证据栏填"无新事件，沿用上周"。

### 打分标准

| 分数 | 监管进展                                                    | 银行竞争威胁                           |
| ---: | ----------------------------------------------------------- | -------------------------------------- |
|  100 | GENIUS Act 最终规则通过，明确支持非银行发行人且不限制 yield sharing | 当月无新银行稳定币企业客户            |
|   75 | 监管进展积极（NPRM 顺利推进或正面听证）；无负面 OCC/FDIC 裁决 | 银行稳定币有讨论但零落地企业客户      |
|   50 | 监管进展缓慢，无正负面事件；GENIUS Act 仍在审议               | 银行稳定币处于试点，企业客户数 <3 个   |
|   25 | OCC/FDIC 出现限制第三方激励或 yield sharing 的草案           | 银行稳定币获得 3-9 个企业试点客户      |
|    0 | GENIUS Act 最终规则明确限制第三方激励或 yield sharing        | 主要商业银行宣布发行稳定币，或企业客户数 ≥10 个 |

两项取较低对应分数。GENIUS Act 出现明确限制条款为单项触发条件，直接判 0 分。

## D7：平台化验证（权重 5%）

### 定义

衡量 CPN、Arc 等平台化产品是否产生可验证的商业收入，以及股票市场筹码结构是否印证平台故事。Other revenue share 是核心指标（季度频率），短期用 short interest 和 CPN 定性进展辅助判断。

### 数据来源

| 指标                           | 来源                       | 更新频率 |
| ------------------------------ | -------------------------- | -------- |
| Other revenue share（÷总收入） | Circle SEC filing          | 季度     |
| CPN TPV 增长（同比）           | Circle earnings disclosure | 季度     |
| Short interest（占流通股比例） | FINRA / 行情平台           | 每周     |
| Implied volatility（30D IV）   | 期权平台                   | 每周     |

Other revenue share 和 CPN TPV 仅季度财报后更新；非财报周沿用上季度值，证据栏标注"沿用 Qx"。

### 打分标准

| 分数 | Other revenue share | CPN TPV 同比增速 | Short interest | 30D IV |
| ---: | ------------------- | ---------------- | -------------- | ------ |
|  100 | ≥10%                | ≥50%             | <5%            | ≤30%   |
|   75 | 7-10%               | 20-50%           | 5-10%          | 30-40% |
|   50 | 5-7%                | 0-20% 或暂未披露 | 10-15%         | 40-50% |
|   25 | 3-5% 或环比下降     | 无增长，仅活跃量 | 15-25%         | 50-65% |
|    0 | <3%                 | 无增长且管理层回避披露 | ≥25%      | >65%   |

CPN TPV 未披露时，该分项按 50 分处理（不按 0 分），但在证据栏标注 `missing_info`。Other revenue share 是主锚。

## 评分异常处理规则

### 触发条件

两名分析师对同一周数据独立打分后，**任意单一维度得分差 ≥15 分，或总分差 ≥10 分**，触发对账程序。

### 对账流程

| 步骤 | 操作                                                                 | 时限    |
| ---- | -------------------------------------------------------------------- | ------- |
| 1    | 双方各自输出"证据列 + 分数档位对照表"，不交换分数，先交换原始证据    | 1 小时内 |
| 2    | 对比具体数值：是否使用了相同来源、相同快照日期、相同计算口径         | 当天内  |
| 3    | 若数值一致但档位判断不同，按本文件阈值逐条核对，以文件规则为准       | 当天内  |
| 4    | 若数值本身不同（来源差异），按来源优先级裁定：SEC filing > Circle 官方 > DefiLlama > CoinGecko > Dune | 当天内 |
| 5    | 裁定后更新 `04-competition-dashboard.md`，并在周度记录模板的 missing_info 行注明差异原因 | 次日复盘前 |

### 不允许的做法

- 不允许取两人分数的平均值，必须根据规则裁定一个答案。
- 不允许因"整体感觉相近"而跳过逐维度对账。
- 数据来源争议未解决前，对应维度填 `missing_info`，不用任何一方的主观判断分数。

### 持续差异记录

连续 3 周出现同一维度分数差 ≥15 分，说明该维度阈值定义不够清晰，需更新本文件阈值，并记录修改版本号和修改原因。
