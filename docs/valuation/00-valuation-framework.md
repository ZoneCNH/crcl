# 估值框架

## 结论

CRCL 不能只看 P/S。

P/S 会高估储备收入质量，因为储备收入要先支付渠道分成。
更好的估值锚是 Market cap / RLDC、Market cap / net reserve economics、Other revenue multiple。

multiple 是估值倍数，白话就是市场愿意为一美元收入或利润付多少钱。

## 三段式估值

```text
基础盘 = average USDC circulation × reserve return rate × RLDC margin
期权盘 = CPN + Arc + enterprise settlement + API + other revenue
折价项 = regulation + distribution dependency + rate pressure + unlock / dilution
```

enterprise settlement 是企业结算，白话就是企业之间用 Circle 网络转账清算。

## 基础盘

基础盘来自储备经济性。

关键公式：

```text
Reserve income = average USDC circulation × reserve return rate
RLDC = total revenue and reserve income - distribution and transaction costs
RLDC margin = RLDC / total revenue and reserve income
```

判断规则：

| 条件                                    | 倍数含义           |
| --------------------------------------- | ------------------ |
| USDC 增长、收益率稳定、RLDC margin 扩张 | 基础盘倍数上修     |
| USDC 增长但收益率下滑                   | 看规模是否抵消利率 |
| USDC 放缓且收益率下滑                   | 双杀，倍数下修     |
| RLDC margin 低于 38%                    | 渠道成本风险上升   |

## 期权盘

期权盘来自平台化收入。

Other revenue 是最直接的财务验证指标。
CPN 和 Arc 是业务线索，但必须转化为可重复收入。

| 阶段     | Other revenue share | 估值处理               |
| -------- | ------------------: | ---------------------- |
| 利差股   |                 <5% | 不给明显平台倍数       |
| 弱平台化 |               5-10% | 给早期期权价值         |
| 中平台化 |              10-15% | 开始拆分估值           |
| 强平台化 |                >15% | 可向支付基础设施股切换 |

## 折价项

| 折价项   | 触发条件                     | 估值动作         |
| -------- | ---------------------------- | ---------------- |
| 监管折价 | 收益分享或第三方激励受限     | 下修 RLDC margin |
| 渠道折价 | Coinbase 议价权增强          | 下修长期利润留存 |
| 利率折价 | 储备收益率下行且 USDC 不增长 | 下修收入         |
| 竞争折价 | USDC 市占率连续下降          | 下修增长率       |
| 筹码折价 | 解禁、增发、内部人减持       | 下修短期风险承受 |

## 估值锚

| 指标                               | 用途                         | 局限             |
| ---------------------------------- | ---------------------------- | ---------------- |
| Market cap / USDC circulation      | 判断市场对每美元 USDC 的定价 | 忽略利率和成本   |
| Market cap / RLDC                  | 判断扣分销后的收入定价       | 仍不是利润       |
| EV / Adjusted EBITDA               | 传统盈利估值                 | 受一次性调整影响 |
| Market cap / net reserve economics | 更贴近储备业务价值           | 需要季度手动计算 |
| Other revenue multiple             | 平台化收入估值               | 需要可重复性验证 |

EV 是 enterprise value，白话就是公司市值加净债务后的企业价值。

## 仓位管理规则

仓位管理与情景判断分离。情景决定研究结论，仓位决定操作幅度。

### 情景与仓位上限映射

| 当前情景 | 建议仓位上限 | 前提条件 |
| -------- | ------------ | -------- |
| Bull | 满仓（100%） | 五项估值切换条件无一缺失，competition score >75 |
| Base | 标准仓位（60-80%） | 基础框架成立，无 P0 风险触发 |
| Bear | 轻仓或观察（20-40%） | 降级但信用层未失效 |
| 信用层触发（C-TRIGGER 任一） | 清仓或极轻仓（0-10%） | 优先于情景判断，立即执行 |

仓位上限是可持有的最高比例，不是建议买入目标比例。
实际仓位可低于上限，根据以下加减因子调整。

### 仓位加减因子

| 因子 | 方向 | 幅度 | 触发条件 |
| ---- | ---- | ---- | -------- |
| USDC 增长加速 | 加仓 | +5-10 ppts | 平均 USDC 同比 + 环比同时超预期 |
| RLDC margin 扩张 | 加仓 | +5 ppts | 连续 2 季超 40% 且无一次性因素 |
| Other revenue 超指引 | 加仓 | +5-10 ppts | 超 FY2026 1.5-1.7 亿指引上沿 |
| competition score >75 | 加仓 | +5 ppts | 竞争结构确认强化 |
| 降息超预期传导 | 减仓 | -10 ppts | Reserve return rate 环比下超 50 bps |
| RLDC margin 低于 38% | 减仓 | -15 ppts | 单季触发，持续则进一步降仓 |
| 筹码 P2 信号（任一） | 减仓 | -5-10 ppts | 大股东减持>5%、做空>15%、解禁集中 |
| competition score <40 | 减仓 | -10-15 ppts | 竞争结构恶化 |
| 监管 P0 风险 | 减仓 | -20 ppts 至触发上限 | 收益分享或激励被限制 |

筹码 P2 信号只调仓位，不切换情景框架，不修改估值倍数。

### 仓位变动纪律

1. 仓位调整必须有触发理由，不能基于股价涨跌本身。
2. 加仓必须在情景未降级的前提下执行。
3. 减仓触发后，不因短期反弹立即恢复仓位；需等下一财报或矩阵信号重新确认。
4. 信用层触发后，不在信用层解除前增仓。

信用层解除条件见 `risk/02-failure-conditions.md`。

## 可比公司分析

可比公司不是定价锚，是倍数校准参照。
用于判断 Market cap / RLDC 倍数区间是否脱离市场定价范围。

### 核心可比公司

| 公司 | 业务类型 | 核心指标 | 为何与 CRCL 可比 | 为何不完全可比 |
| ---- | -------- | -------- | ---------------- | -------------- |
| Visa | 支付网络 | P/E 30-35×、EV/FCF 约 30× | Bull case 倍数上限参考（支付基础设施溢价） | 规模、网络稳定性和盈利能见度远高于 CRCL |
| Mastercard | 支付网络 | P/E 35-40×、EV/FCF 约 32× | Bull case 倍数上限参考（与 Visa 同） | 同上 |
| Coinbase | 加密金融 | P/S 波动大，EV/EBITDA 15-30×（周期依赖） | 加密行业贝塔、合规路径、监管关系类似 | 交易所模式，收入结构与 CRCL 不同 |
| PayPal | 支付 + 金融服务 | P/E 15-20×、EV/EBITDA 10-15× | Base case 倍数参考（成熟支付公司） | 业务复杂度和竞争格局不同 |
| Stripe（私有） | 支付 API | 估值倍数 25-40×（P/S），上市前参考 | 企业支付和 API 商业化路径类似 | 数据不公开，倍数区间宽，参考价值有限 |

### 倍数校准规则

```text
CRCL 当前 Market cap / RLDC 倍数适用范围：

Bear（8-12×）：
  低于 PayPal EV/EBITDA 下沿
  平台化叙事失效，纯利差股定价
  类比：高波动金融公司（BDC、mREIT 区间）

Base（15-22×）：
  接近 PayPal/Coinbase 中间段
  利差股 + 平台期权，合规路径价值
  类比：早期阶段金融科技，有增长但叙事未完全验证

Bull（25-40×）：
  接近 Stripe 私募估值下沿、Visa/Mastercard 低端
  平台化初步验证，支付基础设施重定价
  类比：成长期支付基础设施，多收入流已出现
```

### 可比公司数据更新频率

- Visa / Mastercard：季度财报后更新倍数（约 1 月、4 月、7 月、10 月）
- Coinbase：季度财报后更新（与 CRCL 同期）
- PayPal：季度财报后更新
- Stripe：仅融资轮时更新，参考价值有限

### 使用规则

1. 可比公司倍数只用于校准 CRCL 倍数区间是否合理，不直接套用。
2. Visa / Mastercard 的高倍数仅适用于 Bull case，CRCL 未达到可比规模前不上调倍数区间。
3. Coinbase 倍数受加密周期影响大，只在判断行业贝塔时参考，不用于基本面估值。
4. 每季度框架自检时，核对一次可比公司倍数，记录变化。

## 安全边际公式

```text
估值安全边际 =
  净储备经济性增长
  + Other revenue 增长
  - 利率压力
  - 分销成本压力
  - 监管折价
  - 筹码压力
```

安全边际不是精确数字。
它用于判断风险收益是否还站得住。

## IPO 后流动性折价与溢价

IPO 后短期内存在结构性流动性变化，不影响基本面估值，但影响合理价格区间的判断。

### IPO 后流动性阶段划分

| 阶段 | 时间范围 | 流动性特征 | 对估值的影响 |
| ---- | -------- | -------- | ------------ |
| IPO 初期 | IPO 后 0-30 天 | 承销商稳价机制，流动性人工支撑 | 价格不完全反映自由市场，目标价参考价值有限 |
| 锁定期内 | IPO 后至锁定期到期（通常 6 个月） | 大量浮动筹码被锁定，流通盘小 | 交易量有限，价格可能偏离基本面 |
| 锁定期解禁后 | 解禁后 30-90 天 | 潜在供应大幅增加，流动性提升 | 短期抛压风险，但流动性溢价可能上升 |
| 流动性成熟期 | IPO 后 6-18 个月 | 筹码分布趋于稳定，机构持仓清晰 | 估值折价逐步消除，目标价可靠性提升 |

### IPO 折价 / 溢价的量化参考

IPO 后小盘股、高成长股常见的流动性调整：

```text
IPO 折价（相对成熟可比公司）：通常 10-20%，来自不确定性溢价
锁定期内溢价（筹码被锁）：价格可能短期高于内在价值，因供应受限
解禁冲击折价：解禁规模 >10% 自由流通量时，短期下行风险约 5-15%
流动性成熟溢价：机构覆盖增加后，估值折价逐步收窄
```

这些范围是历史参考，不是 CRCL 专用精确数字。

### 使用规则

1. IPO 后 6 个月内：目标价计算结果需叠加 10-15% IPO 不确定性折扣后才用于仓位判断。
2. 锁定期解禁前 30 天：目标价有效，但仓位上限额外压低 5-10 ppts（见 valuation/00 仓位加减因子表）。
3. 流动性成熟期（IPO 后 12 个月+）：折价逐步消除，按正常目标价框架执行。
4. 若 CRCL 进行二次发行（follow-on offering），短期重新进入 IPO 折价期逻辑。

## 估值切换条件

CRCL 从利差股切换到支付基础设施股，需要同时满足：

1. Other revenue 占比超过 15%。
2. CPN 或 Arc 披露可重复收入贡献。
3. RLDC margin 没有因为平台扩张明显下滑。
4. USDC 市占率稳定或上升。
5. 监管没有限制核心分销激励。

缺一项，只能给期权价值，不能完全切换估值框架。

## 目标价定量模型

目标价不是预测，是把情景假设翻译成数字的工具。
目标价用于判断当前价格是否已经反映了 Base case，以及 Bull/Bear 的赔率比。

### 基础盘估值（主锚：Market cap / RLDC）

```text
步骤一：预估年化 RLDC

  年化 RLDC = 平均 USDC 流通量
            × 储备收益率假设
            × RLDC margin 假设
            + Other revenue 年化
```

情景参数表（以 FY2026 为基准年）：

| 参数 | Bear | Base | Bull |
| ---- | ---- | ---- | ---- |
| 平均 USDC 流通量（亿美元） | 700 | 850 | 1,050 |
| 储备收益率 | 3.2% | 3.6% | 4.0% |
| RLDC margin（÷储备收入） | 38% | 41% | 43% |
| Other revenue（亿美元） | 0.8 | 1.6 | 2.5 |
| 年化 RLDC（亿美元）粗算 | ~8.5 | ~12.6 | ~18.6 |

年化 RLDC 粗算公式：`USDC × 收益率 × margin + Other revenue`
以 Base 为例：850 × 3.6% × 41% + 1.6 ≈ 12.6 亿美元

### 步骤二：确定 Market cap / RLDC 倍数区间

| 情景 | 倍数区间 | 依据 |
| ---- | -------- | ---- |
| Bear | 8-12× | 平台化叙事失效，按纯利差股定价，类比高波动金融公司 |
| Base | 15-22× | 利差股 + 早期平台期权，参考 Coinbase 等加密金融公司 |
| Bull | 25-40× | 平台化初步验证，向支付基础设施股重定价，参考 Visa/MasterCard 低端倍数 |

倍数不是固定数字。倍数区间基于当前框架，需每季度根据矩阵结果和市场环境重新校准。

### 步骤三：计算目标市值区间

```text
目标市值 = 年化 RLDC × Market cap / RLDC 倍数

Bear target：8.5 亿 × 8-12× = 68-102 亿美元
Base target：12.6 亿 × 15-22× = 189-277 亿美元
Bull target：18.6 亿 × 25-40× = 465-744 亿美元
```

目标市值除以当前稀释股数即得每股目标价。
稀释股数应使用最新 10-Q 中的 diluted shares outstanding，不用基本股数。

### 步骤四：安全边际校验

```text
安全边际 = （目标价中值 - 当前股价）/ 当前股价

Base case 安全边际 <20%：估值偏贵，不宜加仓
Base case 安全边际 20-40%：合理进入区间
Base case 安全边际 >40%：显著低估，结合仓位规则加仓
```

### 模型更新触发条件

| 触发事件 | 需重算的参数 |
| -------- | ------------ |
| 季度财报发布 | 平均 USDC 流通量、RLDC margin、Other revenue |
| 美联储利率决议 | 储备收益率假设 |
| 竞争结构重大变化（competition score 越档） | 倍数区间 |
| 情景判断升降级 | 选用情景和对应倍数 |
| 信用层 C-TRIGGER 触发 | 暂停目标价，等信用层恢复后重算 |

## 利率敏感度矩阵

储备收益率对利率的传导路径：美联储基准利率 → 5Y Treasury → USDXX 基金实际收益率 → Circle 储备收益率。
说明：spread 约 50bp，来自管理费、托管费、流动性缓冲成本。下表首行是历史基准样本（5Y Treasury ~4.3%，储备收益率 ~3.8%），运行时必须替换为最新 Treasury / USDXX 数据。

### 储备驱动 RLDC 敏感度（不含 Other revenue）

计算方式：USDC流通量 × 储备收益率 × RLDC margin（41%，Base 口径）

| 美联储基准利率 | 5Y Treasury 约值 | 储备收益率约值 | USDC 700亿 RLDC（亿） | USDC 850亿 RLDC（亿） | USDC 1,050亿 RLDC（亿） |
| -------------- | ---------------- | -------------- | --------------------- | --------------------- | ----------------------- |
| 4.25-4.50%（历史基准） | 4.3% | 3.8% | 10.9 | 13.2 | 16.4 |
| -50bp（3.75-4.00%） | 3.8% | 3.3% | 9.5 | 11.5 | 14.2 |
| -100bp（3.25-3.50%） | 3.3% | 2.8% | 8.0 | 9.8 | 12.1 |
| -150bp（2.75-3.00%） | 2.8% | 2.3% | 6.6 | 8.0 | 9.9 |
| -200bp（2.25-2.50%） | 2.3% | 1.8% | 5.2 | 6.3 | 7.8 |

三列 USDC 流通量对应 Bear（700亿）/ Base（850亿）/ Bull（1,050亿）情景参数，与 `01-scenario-model.md` 保持一致。

全部 RLDC 需在上表基础上加 Other revenue（Bear 0.8亿 / Base 1.6亿 / Bull 2.5亿）。

### 降息对目标价的双重影响

| 降息幅度 | 储备收益率影响 | USDC 规模补偿条件 | 对目标价方向 |
| -------- | -------------- | ----------------- | ------------ |
| -50bp | 储备 RLDC 约下降 10-12% | USDC 同比增长 ≥15% 可大致抵消 | 中性至轻微负向 |
| -100bp | 储备 RLDC 约下降 20-25% | USDC 同比增长 ≥30% 才能抵消 | 负向，需重算 Base |
| -150bp | 储备 RLDC 约下降 30-35% | 难以通过规模完全抵消 | 明显负向，Bear 概率上升 |
| -200bp | 储备 RLDC 约下降 40-50% | 规模无法弥补，倍数需下修 | 强烈负向，考虑 Bear 情景切换 |

降息触发参数重算规则：每次美联储利率决议后，更新"储备收益率假设"一行，重跑目标价计算模板。RLDC margin 假设维持不变，除非渠道分成同步发生变化。

来源：spread 50bp 基于历史样本储备收益率（3.8%）与 5Y Treasury（4.3%）差值推算；RLDC margin 41% 来自 FY2025 全年实际数据（`framework/01-business-model.md`）。运行时重算目标价时，必须用最新利率和最新储备收益率覆盖历史样本。

## 倍数落点规则（区间→点估值）

解决"倍数区间太宽、落点不明"的问题。以下规则在区间内选取具体倍数用于目标价中性值计算。

### Base 情景倍数落点（区间 15-22×）

| 竞争评分（competition score） | 选用倍数 | 逻辑 |
| ----------------------------- | -------- | ---- |
| ＞65 分 | 22× | 竞争结构强化，期权价值可信，取区间高端 |
| 55-65 分 | 18× | 中性竞争态势，取区间中值 |
| ＜55 分 | 15× | 竞争结构弱化，只取区间低端 |

### Bear 情景倍数落点（区间 8-12×）

| RLDC margin（主口径） | 选用倍数 | 逻辑 |
| --------------------- | -------- | ---- |
| ≥38% | 12× | 利润留存尚可，取区间高端 |
| 35-38% | 10× | 留存边界，取区间中值 |
| ＜35% | 8× | 渠道成本侵蚀严重，取区间低端 |

### Bull 情景倍数落点（区间 25-40×）

| 平台化验证等级 | 选用倍数 | 逻辑 |
| -------------- | -------- | ---- |
| 中验证及以上 | 35× | 平台化有财务证据，向支付基础设施股靠拢 |
| 弱验证 | 28× | 有早期迹象但收入未充分拆分，保守处理 |
| 未验证 | 25× | 纯叙事期权，取区间低端 |

### 使用规则

1. 每次计算目标价时，必须在模板中填写"倍数选用依据"，说明根据哪条规则选点。
2. 竞争评分来自 `metrics/04-competition-dashboard.md`，每季度财报后更新。
3. 平台化验证等级来自 `framework/04-platform-option.md`，每季度财报后更新。
4. 规则输入发生变化（如 competition score 越档）时，下次重算时自动采用新规则，不追溯修改已记录的历史目标价。

来源：区间定义参见本文件步骤二倍数区间表；竞争评分说明参见 `metrics/04-competition-dashboard.md`；平台化验证阈值参见 `framework/04-platform-option.md`。

## 稀释股数版本记录

目标价计算必须使用上市后稀释股数，不能使用基本股数。Circle 于 2025 年完成 IPO，稀释股数以上市后最新 10-Q filing 为准。

### 版本记录格式

每次目标价计算必须记录以下稀释股数信息：

```text
稀释股数记录：
  计算日期：
  使用稀释股数：___亿股
  股数来源：S-1（IPO招股书）/ 10-Q（Qx YYYY）/ 10-K（FYYYY）
  Filing 版本日期：
  是否含期权/RSU 等稀释项：是 / 否，说明：
  与上次记录的差异：___百万股（原因：增发 / 解禁 / 回购 / 无变化）
```

### 稀释股数运行时刷新记录

| 项目 | 记录 |
| ---- | ---- |
| 使用稀释股数 | 运行时从最新 10-Q / 10-K / S-1 / 424B 刷新后填写 |
| 参考来源 | Circle IPO S-1 + 最新 10-Q / 10-K / 424B |
| 上次确认日期 | 每季度财报后更新 |
| 注意事项 | IPO 后锁定期结束可能带来增量稀释；Form 4 集中披露时需复核 |

### 更新触发条件

| 触发事件 | 动作 |
| -------- | ---- |
| 季度 10-Q 发布 | 核对 diluted shares outstanding，更新记录 |
| 内部人 Form 4 集中披露 | 确认是否影响全稀释股数 |
| 二次增发或可转债转股 | 立即更新，重算所有情景目标价 |
| 股票回购 | 确认规模，更新并注明对目标价的影响方向 |

来源：稀释股数应取自 SEC EDGAR 最新 10-Q "Diluted weighted-average shares outstanding" 行；IPO 参考文件为 Circle S-1（2025年）及后续修订版本。
