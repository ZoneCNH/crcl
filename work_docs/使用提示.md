# CRCL Codex 使用提示

本文件用于在当前 CRCL 项目中直接驱动 Codex 运行日监、监控巡检、周报、财报复核、估值/仓位和框架自检。

使用方式：在本仓库根目录打开 Codex，然后复制下面对应提示词。Codex 应读取 `AGENTS.md`、`README.md`、`work_docs/subagent-decision-workflows.md` 以及相关 `docs/` 规则，按项目框架给出最终判断，而不是只告诉你该运行什么命令。

## 通用总控提示词

```text
你在当前仓库内工作。

请按本项目 AGENTS.md 和 CRCL 研究框架完成一次复核：

任务类型：<日监 / 监控巡检 / 周度复盘 / 财报复核 / 估值仓位 / 框架自检 / 监管事件 / 竞争复盘 / 平台化复核>
数据策略：<允许刷新当前/最新数据 / 只读已有数据库，不联网，必须 --no-collect>
当前仓位：<如 50%，没有就写无>
保存要求：除非我明确说“不保存”，请保存最终产物到 work_docs/agent_runs/。

请自动判断 workflow，必要时运行项目 agent。需要同一批证据时，先由 collector 生成或确认对应 decision-pack / agent-run evidence packet，避免不同 agent 自己刷新出不同口径。

必须先做 collector 数据收集编排，再做 data-quality 和 source 前置检查，再派发财务、监管、竞争、平台化、风险或 autoresearch agent。最后由主线程或 risk-decision 收口。

最终用中文输出：
1. 结论：增强 / 降级 / 观察，或 Bull / Base / Bear，或 keep / revise / defer / reject。
2. collector 是否完成采集，以及 data-quality 是否阻断。
3. 关键证据：指标值、数据日期、来源。
4. missing_info。
5. 反证 / 解除条件。
6. 下一步动作或下次复盘触发。
7. 已保存的 work_docs/agent_runs/*.md 路径；如果我明确说不保存，则说明未保存。

不要输出个性化投资建议。不要只写“利好”“利空”。
```

## 日监

允许刷新当前数据并保存：

```text
做一次 CRCL 日监，自动调用项目 agent，允许刷新当前数据，保存产物，最后直接给我结论、collector 采集结果、data-quality 是否阻断、P0/P1 信号、missing_info、反证和下一步动作。
```

## 监控巡检

允许刷新并保存到监控目录：

```text
做一次 CRCL 自动化监控巡检，自动调用项目 agent，允许刷新当前数据，保存到 work_docs/monitoring/。请先由 collector 生成 monitoring evidence packet，再由 data-quality、source、regulatory 和 monitor guard 判断数据源失败、P0 missing_info、过期指标、状态页异常和紧急事件。最后给我正常 / 警戒 / 紧急、阻断项、补采动作、解除条件和已保存路径。
```

只读巡检：

```text
做一次 CRCL 自动化监控巡检，只读已有数据库，不联网，必须 --no-collect，自动调用项目 agent，保存到 work_docs/monitoring/，最后给我数据健康是否阻断、失败源、missing_info 和下一次检查触发。
```

只读已有数据库：

```text
做一次 CRCL 日监，只读已有数据库，不联网，必须 --no-collect，自动调用项目 agent，保存产物，最后给我观察 / 增强 / 降级结论和阻断项。
```

## 周度复盘

```text
做一次 CRCL 周度复盘，自动调用项目 agent，允许刷新本周最新数据，保存产物。请先由 collector 收集同一批证据，再给出 D1-D7 竞争评分、competition score、data-quality/source 阻断项、关键证据、missing_info、反证/解除条件和下周触发条件。
```

只读复盘：

```text
做一次 CRCL 周度复盘，只读已有数据库，不刷新外部数据，必须 --no-collect，自动调用项目 agent，保存产物。最后直接给结论、评分和下一步。
```

## 财报 / 季报复核

```text
做一次 CRCL 财报复核，自动调用项目 agent，允许刷新最新 SEC、市场、利率和事件数据，保存产物。请按项目财报框架区分事实数字、叙事变化、missing_info 和反证，不要从单条新闻直接给仓位动作。
```

财报 T+0 只看事实数字：

```text
做一次 CRCL 财报 T+0 复核，只读或刷新由你按当前数据可得性判断，自动调用项目 agent，保存产物。T+0 只填事实数字和 data-quality/source 问题，不输出仓位动作。
```

## 估值 / 仓位 / 交易决策

把 `<当前仓位>` 改成真实仓位：

```text
做一次 CRCL 估值和仓位决策，当前仓位 <当前仓位>%，自动调用项目 agent，允许刷新最新数据，保存产物。请输出 Bull/Base/Bear、估值区间、仓位上限、加减仓动作、data-quality/source 阻断、关键证据、missing_info、反证和下一次复盘触发。不要输出个性化投资建议。
```

只读已有数据库：

```text
做一次 CRCL 估值和仓位决策，当前仓位 <当前仓位>%，只读已有数据库，不联网，必须 --no-collect，自动调用项目 agent，保存产物。最后直接给 Bull/Base/Bear 和动作边界。
```

## 框架自检

```text
做一次 CRCL 框架自检，自动调用项目 agent，允许刷新当前数据，保存产物。请按 autoresearch / keep-revise-defer-reject 框架判断本轮是 keep、revise、defer 还是 reject，并列出 data-quality/source 阻断、需要补采的数据、文档修改建议和下次复盘触发。不要直接改文档，除非我另行要求。
```

只读框架自检：

```text
做一次 CRCL 框架自检，只读已有数据库，不联网，必须 --no-collect，自动调用项目 agent，保存产物。请判断 keep / revise / defer / reject，并说明证据、missing_info 和建议。
```

## 监管事件复核

```text
复核 CRCL 当前监管事件，自动调用项目 agent，允许刷新最新监管和市场事件数据，保存产物。请先做 collector 数据收集和 data-quality/source 检查，再给出事件等级、影响路径、关键证据、missing_info、反证/解除条件和下一步动作。
```

## 竞争复盘

```text
做一次 CRCL 竞争复盘，自动调用项目 agent，允许刷新最新数据，保存产物。请按 D1-D7 逐项评分，不要跳过 data-quality/source 前置检查；最后给 competition score、主要变化、反证和下次观察指标。
```

## 平台化复核

```text
做一次 CRCL 平台化复核，自动调用项目 agent，允许刷新最新数据，保存产物。请重点检查 Arc/mainnet、支付网络、开发者采用、合作伙伴和平台 option 证据；最后给平台化结论、missing_info、反证和下一步触发。
```

## 命令兜底

通常让 Codex 自动选择命令即可。需要人工指定时使用：

```bash
cargo run --release -- agent-run --workflow daily-monitor --save
cargo run --release -- agent-run --workflow monitoring --save --out-dir work_docs/monitoring
cargo run --release -- agent-run --workflow weekly-review --save
cargo run --release -- agent-run --workflow quarterly-earnings --save
cargo run --release -- agent-run --workflow valuation-decision --current-position-pct 50 --save
cargo run --release -- agent-run --workflow framework-review --save
```

只读已有数据库时追加 `--no-collect`：

```bash
cargo run --release -- agent-run --workflow weekly-review --no-collect --save
cargo run --release -- agent-run --workflow valuation-decision --no-collect --current-position-pct 50 --save
```

只生成证据包、不做最终判断时使用：

```bash
cargo run --release -- decision-pack --workflow daily-monitor
cargo run --release -- decision-pack --workflow monitoring
cargo run --release -- decision-pack --workflow weekly-review
cargo run --release -- decision-pack --workflow quarterly-earnings
cargo run --release -- decision-pack --workflow valuation-decision
cargo run --release -- decision-pack --workflow framework-review
```

脚本消费 JSON 时直接追加 `--format json`，Rust 命令不会输出 npm 脚本头。

## 输出检查清单

每次 Codex 最终回答都应包含：

- 结论：增强 / 降级 / 观察，或 Bull / Base / Bear，或 keep / revise / defer / reject。
- collector 是否完成采集，data-quality 是否阻断，source 是否可信。
- 关键证据：指标值、数据日期、来源。
- missing_info，尤其是 P0/P1 缺口。
- 反证 / 解除条件。
- 下一步动作或下次复盘触发。
- 已保存的 `work_docs/agent_runs/*.md` 或定时分类目录 `work_docs/<workflow>/*.md` 路径；如果明确不保存，则说明未保存。

如果 data-quality 或 source 阻断未解除，不能强行输出增强、keep 或乐观仓位动作。
